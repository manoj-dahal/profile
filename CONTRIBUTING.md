# 🤝 Contributing to Manoj Dahal Portfolio

Thank you for your interest in contributing! This document outlines the process for contributing to this portfolio.

## 🐛 Reporting Bugs

Found a bug? Please open an issue using the [bug report template](.github/ISSUE_TEMPLATE/bug_report.md).

When filing a bug report, please include:
- A clear, descriptive title
- Steps to reproduce the issue
- Expected vs. actual behavior
- Screenshots (if applicable)
- Browser, device, and OS information

## ✨ Suggesting Features

Have an idea? Open a [feature request](.github/ISSUE_TEMPLATE/feature_request.md) and tell us:
- What the feature does
- Why it would be useful
- How you envision it working

## 🔧 Submitting Pull Requests

1. **Fork** the repository
2. **Create a branch** for your feature: `git checkout -b feature/amazing-idea`
3. **Make your changes** following the code style
4. **Test** in multiple browsers and screen sizes
5. **Validate** by running `npm run validate`
6. **Commit** with a descriptive message
7. **Push** to your branch
8. **Open a Pull Request** using the [PR template](.github/pull_request_template.md)

## 📝 Code Style

- **HTML**: Semantic markup, 2-space indent
- **CSS**: Follow BEM-like naming where applicable
- **JS**: ES6+, prefer `const`/`let`, use arrow functions, document with JSDoc
- **File names**: `kebab-case` for files, `camelCase` for JS variables

## 🧪 Testing

Before submitting a PR, verify:
- [ ] Works in Chrome, Firefox, Safari
- [ ] Works on desktop, tablet, and mobile
- [ ] No console errors
- [ ] All animations are smooth (60fps target)
- [ ] `npm run validate` passes
