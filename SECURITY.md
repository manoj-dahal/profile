# 🔒 Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 1.0.x   | ✅ Active support  |
| < 1.0   | ❌ No longer supported |

## Reporting a Vulnerability

If you discover a security vulnerability in this portfolio, please report it privately.

**Please do NOT open a public GitHub issue for security vulnerabilities.**

### How to Report

Email: **info@manoj-dahal.com.np**

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### What to Expect

- **Acknowledgment** within 48 hours
- **Status update** within 7 days
- **Fix timeline** depends on severity:
  - Critical: 24-48 hours
  - High: 1 week
  - Medium: 2 weeks
  - Low: Next release

## Security Best Practices

This portfolio follows these practices:
- ✅ HTTPS-only deployment
- ✅ Strict CSP headers (recommended for production)
- ✅ No third-party tracking scripts
- ✅ No data collection
- ✅ All assets served from same origin
- ✅ Dependencies kept minimal (zero runtime deps)
